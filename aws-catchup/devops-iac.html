1. Terraform + AWS

Updated DevOps SA Pro

License & Ecosystem Upheaval (2023–2025)

Aug 2023 — HashiCorp BSL License Change

Terraform switched from MPL 2.0 to BSL 1.1 (Business Source License). This restricts competitive commercial use — you cannot embed Terraform in a competing product. No operational impact for teams using Terraform internally or with HCP Terraform.

Sep 2023 — OpenTofu Fork

The Linux Foundation accepted OpenTofu as an official project, maintaining the MPL 2.0 license. CLI compatible with Terraform. Unique features include:

  • State Encryption (v1.7) — encrypt state at rest natively
  • OCI Registry support for modules and providers
  • Native S3 locking without DynamoDB table
Feb 2025 — IBM Acquired HashiCorp ($6.4B)

IBM completed the acquisition. BSL license remains unchanged. No immediate product impact.

Apr 2024 — Terraform Cloud Renamed HCP Terraform

Terraform Cloud rebranded to HCP Terraform (HashiCorp Cloud Platform). Legacy free tier ended March 2026.

HCP Terraform Pricing Tiers (Post-March 2026)

Tier Resources Cost Key Features
Free Up to 500 $0 Remote state, basic runs
Essentials 500+ $0.10/resource/mo Team management, SSO
Standard 500+ $0.47/resource/mo Policy as code, drift detection
Premium 500+ $0.99/resource/mo Audit logging, concurrent runs, custom agents

AWS Provider v5

  • Breaking changes — EC2-Classic resource removals, deprecated argument cleanups
  • Requires Terraform ≥ 1.0
  • New default tags propagation

Best Practices

  • Use remote backends (S3 + DynamoDB for locking, or HCP Terraform)
  • Always commit .terraform.lock.hcl to version control
  • Separate workspaces per environment (dev/staging/prod)
  • Keep modules small and composable
  • Use pre-commit hooks: terraform fmt, terraform validate, tflint, Checkov
  • Never commit terraform.tfstate or secrets to version control
EXAM TIP: IaC is 17% of DOP-C02. Know the decision criteria for Terraform vs CDK vs CloudFormation. Understand BSL license implications and when OpenTofu is a valid alternative.

2. AWS CDK

Major Update DevOps SA Pro
CDK v1 reached end of support on June 1, 2023. All new development must use CDK v2. Migration is required for existing v1 projects.

CDK v2 — What Changed

  • Single package model: all stable constructs consolidated into aws-cdk-lib. No more installing individual service packages (e.g., @aws-cdk/aws-s3).
  • Experimental constructs moved to separate alpha packages
  • cdk watch for faster iterative development — auto-deploys on file changes

Three Construct Levels

Level Name Description
L1 CFN Resources Auto-generated 1:1 CloudFormation mapping. Prefixed with Cfn. Full control, no defaults.
L2 Curated Constructs AWS-curated with sensible defaults and best practices. Most commonly used. E.g., s3.Bucket.
L3 Patterns High-level combinations of multiple resources. E.g., ApplicationLoadBalancedFargateService.

CDK Ecosystem

  • CDK for Terraform (CDKtf): define Terraform infrastructure using programming languages (TypeScript, Python, Java, C#, Go)
  • CDK Pipelines: self-mutating CI/CD pipelines — the pipeline updates itself when you push changes to the pipeline definition
EXAM TIP: Understand the CDK v2 single-package model (aws-cdk-lib), the three construct levels (L1/L2/L3), and CDK Pipelines for self-mutating deployments. CDK synth produces CloudFormation templates.

3. CloudFormation

Updated DevOps SA Pro

Key New Capabilities (2022–2025)

Feb 2022 — Hooks (GA)

Pre-create, pre-update, and pre-delete policy enforcement. Run custom validation logic before CloudFormation provisions resources. March 2025: Managed hooks using Lambda + Guard rules — simplified hook authoring.

2022 — Modules

Reusable resource groups published via the CloudFormation Registry. Package common patterns and share across accounts/organization.

2023 — Git Sync

Auto-triggers stack deployments on git push. Supports GitHub, GitLab, and Bitbucket. Enables GitOps workflows natively.

Feb 2024 — IaC Generator (GA)

Generates CloudFormation templates from existing resources. Enables brownfield adoption — bring unmanaged resources under CloudFormation control without recreating them.

Feb 2025 — Stack Refactoring

Move resources between stacks without recreating them. Supports up to 5 stacks in a single refactoring operation via CLI. Enables evolving architectures without downtime.

EXAM TIP: Hooks for governance/compliance enforcement, Git Sync for GitOps patterns, IaC Generator for brownfield adoption, and Stack Refactoring for evolving multi-stack architectures. Know when to use each.

4. CodePipeline V2

Major Update DevOps
V2 pipeline type launched October 2023 and became the default pipeline type in December 2024. V1 pipelines remain functional but lack new features.

V1 vs V2 Comparison

Feature V1 V2
Git tag triggers No Yes
Pipeline variables / parameters No Yes
Execution modes Superseded only Superseded, Queued, Parallel
Trigger filters (branch, file path, PR) No Yes
Source revision overrides / rollback No Yes
Stage-level rollback No Yes (Apr 2024)

Execution Modes Explained

  • Superseded: new execution cancels in-progress execution (V1 default and only mode)
  • Queued: executions queue and run in order — no skipping
  • Parallel: multiple executions run simultaneously — ideal for independent deployments
EXAM TIP: V2 is now the default. Understand all three execution modes and their use cases. Stage-level rollback is a key differentiator — enables reverting a single stage without rerunning the entire pipeline.

5. CodeBuild

Updated DevOps

Key Updates

  • Lambda compute support: faster build startup times using Lambda-based compute — ideal for short, lightweight builds
  • macOS builds: available via Reserved Capacity — enables iOS/macOS CI/CD workflows
  • GitLab / GitLab Self-Managed as source: native integration alongside GitHub, Bitbucket, and CodeCommit

  • Secrets Manager integration: securely reference git credentials and build secrets without embedding them in buildspec files
EXAM TIP: Know when to use Lambda compute (fast, lightweight) vs standard compute (Docker builds, long-running). Docker Server for build caching scenarios.

6. CodeDeploy

Updated DevOps

Zonal Deployments for EC2 (Dec 2023)

  • Deploy one Availability Zone at a time — reduces blast radius
  • Configure minimum healthy instances per AZ to maintain capacity during deployment
  • Requires custom deployment configuration — not available with default deployment configs
  • Enables AZ-aware canary and linear deployment strategies
Key distinction: Zonal deployments complement existing deployment strategies (in-place, blue/green) by adding AZ-level granularity. This is critical for multi-AZ resilience during deployments.

  • All-in-one: source control, CI/CD, issue tracking, cloud development environments
CLOSED to new customers. CodeCatalyst is no longer accepting new sign-ups. Existing customers can continue using the service. For new projects, use the traditional Code* services (CodeCommit, CodeBuild, CodePipeline, CodeDeploy).

AWS CodeCommit

Deprecation REVERSED. CodeCommit was initially marked for deprecation in mid-2024 but was restored to General Availability on November 24, 2025. Git LFS support planned for Q1 2026.
  • Fully managed Git repositories
  • Integrated with IAM for authentication and authorization
  • Supports pull requests, approval rules, and triggers
EXAM TIP: CodeCommit is GA and actively supported. CodeCatalyst is closed to new customers. Know that CodeCommit integrates with CodePipeline, CodeBuild, and EventBridge for CI/CD workflows.

8. ECS Major Updates

Updated DevOps SA Pro

ECS Service Connect (Nov 2022)

  • Built-in service-to-service communication via AWS Cloud Map
  • Envoy sidecar proxy auto-injected into tasks — no manual configuration
  • Round-robin load balancing + outlier detection out of the box
  • No code changes needed — services discover each other by namespace
  • Replaces AWS App Mesh for many ECS service mesh use cases

EBS Volume Integration (Jan 2024)

  • Attach EBS volumes to Fargate and EC2 tasks for storage-intensive workloads
  • Volumes auto-provisioned when task starts, auto-deleted when task stops
  • Enables stateful containers (databases, ML training, data processing) on Fargate

Built-in Blue/Green Deployment (2024)

  • Native canary deployment with automated rollbacks
  • No CodeDeploy intermediary needed — ECS handles blue/green natively
  • Simplifies deployment pipeline architecture

Other Updates

  • Graviton4 support (2025): ~40% better price-performance for ARM-based workloads
  • Classic console deprecated (Dec 2023): new console is the only option
EXAM TIP: Service Connect vs App Mesh — Service Connect is simpler and preferred for ECS-only environments. Know EBS volume support on Fargate for stateful workloads. Built-in blue/green removes the CodeDeploy dependency.

9. EKS Major Updates

Updated DevOps SA Pro

EKS Pod Identity (Nov 2023)

A simpler, more scalable alternative to IRSA (IAM Roles for Service Accounts) for granting AWS permissions to Kubernetes pods.

Feature IRSA Pod Identity
Setup OIDC provider + trust policy per cluster EKS-native service principal
Cross-cluster reuse Separate trust policy per cluster Same role reusable across clusters
Session tags No Yes
Cross-account access Complex (chained assume-role) Native support (June 2025)
IRSA is NOT deprecated. Both IRSA and Pod Identity coexist. Pod Identity is the recommended approach for new workloads, but existing IRSA configurations continue to work.

EKS Auto Mode (Dec 2024)

  • Fully automates compute, storage, and networking management
  • Powered by Karpenter for intelligent node provisioning
  • Pre-configured add-ons (CoreDNS, kube-proxy, VPC CNI, EBS CSI driver)
  • 51–70% TCO reduction compared to self-managed EKS
  • AWS manages node lifecycle, patching, and scaling

EKS Blueprints

  • CDK and Terraform framework for provisioning production-ready EKS clusters
  • Opinionated defaults for networking, observability, security, and add-ons
  • Extensible via add-on patterns
EXAM TIP: Pod Identity vs IRSA is a critical exam topic. Know the comparison table. EKS Auto Mode represents a paradigm shift — understand when fully managed compute is appropriate vs self-managed node groups.
EKS User Guide EKS Pod Identity EKS Auto Mode

10. Lambda Updates

Updated DevOps SA Pro

Major Feature Launches

Mar 2022 — 10 GB Ephemeral Storage

Configurable from 512 MB up to 10 GB in /tmp. Previously fixed at 512 MB.

Apr 2022 — Function URLs

Built-in HTTPS endpoints for Lambda functions. No API Gateway needed for simple HTTP use cases. Supports IAM auth or no auth (public). Not a replacement for API Gateway when you need throttling, request validation, or API keys.

2022 — Graviton2/ARM Support

34% better price-performance compared to x86. Set architecture to arm64. Most runtimes supported.

Nov 2022 — SnapStart (Java)

Caches initialized memory and disk snapshot after init phase. Sub-second cold starts for Java functions.

Apr 2023 — Response Streaming

Progressive response delivery — stream partial results to clients as they become available. Useful for large responses or real-time processing.

Nov 2024 — SnapStart for Python 3.12+ and .NET 8+

Extended SnapStart beyond Java. Same snapshot caching mechanism for sub-second cold starts.

Nov 2025 — Managed Instances

Serverless management model + EC2-style compute. Up to 32 GB RAM and 16 vCPUs. You configure instance type and fleet size. Bridges the gap between Lambda and EC2/ECS.

SnapStart Compatibility

SnapStart is NOT supported with:
  • Node.js runtime
  • Provisioned Concurrency (mutually exclusive)
  • Amazon EFS file system mounts
  • Ephemeral storage greater than 512 MB
EXAM TIP: SnapStart is the answer for cold-start optimization (Java, Python 3.12+, .NET 8+). Function URLs for simple HTTPS without API Gateway. Know the SnapStart incompatibilities — especially Provisioned Concurrency and EFS.
Lambda Developer Guide Lambda SnapStart Function URLs

11. Step Functions Updates

Updated DevOps SA Pro

Distributed Map (Dec 2022)

  • Process up to 10,000 parallel child executions
  • Source data directly from S3 (CSV, JSON, S3 inventory)
  • Redrive failed children only (Nov 2023) — no need to re-run successful executions
  • Ideal for large-scale data processing, ETL, and batch operations

JSONata Support (Nov 2024)

  • Replaces 5 JSONPath fields (InputPath, OutputPath, ResultPath, ResultSelector, Parameters) with just 2: Arguments + Output
  • Built-in date/time, math, and string functions
  • Variables accessible across states
  • Dramatically simplifies data transformation in state machines

Integration Types

Type Scope IAM Use When
Optimized Pre-built for select services (Lambda, DynamoDB, SQS, SNS, ECS, etc.) Auto-generated IAM policies Target service has an optimized integration
AWS SDK 200+ AWS services, 9,000+ API actions Manual IAM policy configuration No optimized integration exists, or you need a specific API action
EXAM TIP: Distributed Map for large-scale parallel processing. Know the difference between Optimized and SDK integrations. JSONata simplifies data transformation but JSONPath is still supported.

12. EventBridge Updates

Updated DevOps SA Pro

EventBridge Pipes (GA 2022–2023)

Point-to-point integrations that connect sources to targets with optional filtering, enrichment, and transformation — without writing Lambda glue code.

Three-step pipeline:
  1. Filter — event pattern matching to process only relevant events
  2. Enrich — call Lambda, API Gateway, API Destination, or Step Functions to augment the event
  3. Transform — reshape the event payload before delivery to the target

Supported sources: DynamoDB Streams, Kinesis Data Streams, SQS, Amazon MSK, Amazon MQ

Supported targets: 14+ services including Lambda, Step Functions, SQS, SNS, Kinesis, ECS, API Gateway, and more

EventBridge Scheduler (2022)

  • One-time or recurring schedules with cron or rate expressions
  • 270+ AWS service targets — invoke nearly any AWS API on a schedule
  • Time zone aware — handles daylight saving transitions
  • Built-in retry policies and dead-letter queue support
EXAM TIP: EventBridge Pipes replaces Lambda glue functions for point-to-point event processing. EventBridge Scheduler replaces many CloudWatch Events / EventBridge rules for scheduled tasks. Know the three-step Pipes model (Filter → Enrich → Transform).
EventBridge User Guide EventBridge Pipes EventBridge Scheduler

13. Systems Manager

Updated Partial Sunset DevOps

Capabilities Closed to New Customers

Closed to new customers (Nov 2025):
  • Incident Manager — incident response and on-call management
  • Change Manager — change request and approval workflows
Existing customers can continue using these services.

Active & Updated Capabilities

  • Fleet Manager: GUI-based fleet management — view and manage instances across accounts/regions
  • Parameter Store: hierarchical configuration and secrets storage — standard (free) and advanced tiers
  • Session Manager: secure shell access without SSH keys, bastion hosts, or open inbound ports
  • Patch Manager: automated patching across OS and application packages
  • Automation: runbook-based operational workflows with approval gates
  • Run Command: remote command execution across managed instances at scale
EXAM TIP: Focus on the active SSM capabilities. Session Manager is a common exam topic for secure instance access. Parameter Store vs Secrets Manager is a classic comparison. Incident Manager and Change Manager are winding down.

14. CloudWatch Updates

Updated DevOps SA Pro

Application Signals (GA 2024)

AWS-native Application Performance Monitoring (APM) — a direct alternative to third-party tools like Datadog, New Relic, and Dynatrace.

  • Auto-instrumented via OpenTelemetry — no code changes required
  • Monitors the four golden signals: latency, throughput, errors, availability
  • SLO definition and tracking — set service level objectives and alert on breaches
  • Service dependency map — automatic visualization of service relationships
  • Supported: Java, Python, Node.js on Lambda, ECS, EKS, and EC2

Internet Monitor (2023)

  • Monitors internet-facing application performance from the end-user perspective
  • Detects ISP and geographic issues impacting availability and performance
  • Integrates with CloudWatch dashboards and alarms

Cross-Account Observability (2022+)

  • Share metrics, logs, and traces across AWS accounts via Observability Access Manager (OAM)
  • Designate monitoring accounts and source accounts
  • No extra cost for shared logs and metrics
  • Essential for multi-account AWS Organizations architectures
EXAM TIP: Application Signals is the new AWS-native APM solution. Know the four golden signals (latency, throughput, errors, availability) and SLO capabilities. Cross-account observability via OAM is critical for multi-account architectures on both SA Pro and DevOps Pro exams.
CloudWatch User Guide Application Signals Cross-Account Observability

15. Resilience Hub

New — Nov 2021 DevOps SA Pro

Core Capabilities

  • Resilience Policies: define RTO (Recovery Time Objective) and RPO (Recovery Point Objective) targets per application
  • Assessment: evaluate applications against the Well-Architected Framework and the Resilience Analysis Framework
  • Dependency discovery: automatically identify infrastructure dependencies and single points of failure
  • Recommendations: actionable suggestions to meet defined RTO/RPO targets
  • Drift detection: alert when infrastructure changes compromise resilience posture

2025 Enhancements

  • GenAI-powered failure mode analysis — AI identifies potential failure scenarios
  • Multi-agent AI engine for automated resilience assessment and remediation suggestions
EXAM TIP: Resilience Hub is the central service for defining and assessing application resilience. Know the relationship between RTO/RPO policies and the assessment capabilities. Pairs with Fault Injection Service for chaos engineering.

16. Fault Injection Service

Updated DevOps
Renamed from AWS Fault Injection Simulator to AWS Fault Injection Service (late 2023).

Experiment Targets

  • EC2: stop/terminate instances, stress CPU/memory/disk/network
  • ECS: stop tasks, drain containers, network fault injection (2025)
  • EKS: terminate pods, node drain, pod network faults
  • Lambda: fault injection via chaos extension (Oct 2024)
  • RDS: failover, reboot instances
  • Networking: disrupt connectivity between AZs, subnets, or to external services

Key Features

  • Scenario Library: pre-built experiment templates for common failure scenarios
  • Stop conditions: CloudWatch alarms that automatically halt experiments if thresholds are breached
  • IAM-controlled blast radius: scope experiments with precise resource targeting
  • Integration with Resilience Hub for validating resilience assessments
EXAM TIP: FIS is the managed chaos engineering service. Know the supported targets and how stop conditions work. Pair with Resilience Hub for a complete resilience testing strategy.
Fault Injection Service User Guide

17. AWS AppConfig (Feature Flags)

Updated DevOps

Key Milestones

Mar 2022 — Feature Flags GA

Native feature flag support — AWS answer to LaunchDarkly. Define flags, target users, and roll out features safely.

Jul 2024 — Advanced Targeting

A/B testing, user segment targeting, and variants. Fine-grained control over who sees which feature.

Architecture

  • Part of AWS Systems Manager
  • Configuration backends: hosted configuration store, S3, Parameter Store, or Secrets Manager
  • Deployment strategies: linear, exponential, all-at-once with automatic rollback
  • CloudWatch alarms integration for safe rollouts — automatic rollback on alarm breach
EXAM TIP: AppConfig Feature Flags = safe gradual rollout of features. “Enable feature for 10% of users, monitor, then expand” = AppConfig. Pairs with CloudWatch alarms for automated rollback. Key DevOps Pro topic for deployment strategies beyond blue/green and canary.

Sidecar pattern support — run logging, monitoring containers alongside main job container.
