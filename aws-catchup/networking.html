1. VPC Lattice

NEW — GA March 2023 SA Pro Networking

Service-to-service communication across VPCs and accounts without VPC peering, Transit Gateway, or PrivateLink. Application-layer networking that abstracts away the underlying network topology.

Key Concepts
  • Service Network — logical boundary that multiple VPCs can join. Controls which services are accessible to which consumers.
  • Services — application units, each with an auto-generated DNS name. Represent workloads you want to expose.
  • Target Groups — EC2 instances, IP addresses, Lambda functions, ECS tasks, EKS pods, or ALBs behind a service.

Auth Model

  • IAM auth policies (SigV4) — similar to resource-based policies, attached to the service network or individual services
  • Fine-grained control: who can call which service, from which VPC, with what identity

Protocols

  • HTTP, HTTPS, gRPC, TCP

Key Benefits

  • Solves overlapping CIDR problems — operates at Layer 7, does not require non-overlapping IP ranges
  • No inter-AZ data transfer charges within Lattice
  • Replaces App Mesh for many service-mesh use cases
  • Weighted routing for blue/green and canary deployments
Architecture Pattern: Central networking account owns the service network. Spoke accounts register their services. Consumer VPCs associate with the service network to discover and call services.
EXAM TIP: "Service-to-service communication across VPCs and accounts" = VPC Lattice. This is distinct from ALB/NLB (which operate within a VPC). Lattice works across VPCs, accounts, and compute types without peering or TGW.

2. AWS Cloud WAN

NEW — GA July 2022 SA Pro Networking

Managed global WAN service that uses the AWS backbone to build and operate a unified global network with a centralized, declarative policy document.

Key Concepts
  • Global Network — top-level container for your entire WAN
  • Core Network — the AWS backbone that interconnects your attachments
  • Segments — isolation domains (e.g., prod, dev, shared-services). Traffic is isolated between segments unless explicitly shared.
  • Attachments — VPCs, VPN connections, Transit Gateways, Direct Connect Gateways connected to the core network
  • Core Network Policy — declarative JSON/YAML document that defines segments, routing, and attachment rules

Key Features

  • Attach existing Transit Gateways for incremental migration from TGW-based architectures
  • Service Insertion (2023–2024) — steer traffic through firewalls, IDS/IPS, or other security appliances
  • Routing Policy (2024) — BGP attribute manipulation, route filtering, and preference control
TGW vs Cloud WAN: Transit Gateway = single-Region hub-and-spoke. Cloud WAN = multi-Region global orchestration with centralized, intent-driven policy.
EXAM TIP: Cloud WAN is policy-based, intent-driven networking. Know how segments provide isolation and when to use Cloud WAN vs TGW. Cloud WAN for multi-Region; TGW for single-Region hub-spoke.
AWS Docs: Cloud WAN User Guide

3. AWS Verified Access

NEW — GA April 2023 SA Pro Networking Security

Zero-trust access to corporate applications without a VPN. Evaluates identity and device posture on every request before granting access.

How It Works

  • Each request is evaluated against access policies — no implicit trust based on network location
  • Uses Cedar policy language (same engine as Verified Permissions)
  • Supports HTTP/HTTPS and TCP/SSH/RDP (Dec 2024 preview)

Integrations

  • Identity Providers: IAM Identity Center, Okta, CyberArk, and other OIDC/SAML providers
  • Device Management: CrowdStrike, Jamf — assess device posture (patched, compliant, managed)
  • WAF: attach AWS WAF web ACLs for additional Layer 7 protection
EXAM TIP: "Replace VPN for corporate app access" or "zero-trust access" = AWS Verified Access. Combines identity + device posture for per-request authorization.
AWS Docs: Verified Access User Guide

4. VPC IPAM

NEW — Dec 2021 SA Pro Networking

Centralized IP address management across AWS accounts and Regions. Plan, track, and monitor IP addresses at scale.

Key Capabilities

  • Organize IP addresses into hierarchical pools with business rules (e.g., per-Region, per-environment)
  • Auto-discover used IP addresses across your organization
  • Supports IPv4 and IPv6 (including ULA fd00::/8 and GUA)
  • BYOIP management — onboard and manage your own IP ranges
  • Sequential Elastic IP allocation — allocate EIPs from specific IPAM pools
  • Organizations integration — enforce IP allocation policies across member accounts
Why It Matters: Before IPAM, IP planning was typically done in spreadsheets, leading to overlaps and conflicts. IPAM provides automated, enforceable IP governance at scale.
AWS Docs: VPC IPAM User Guide

5. Route 53 Updates

UPDATED SA Pro Networking

Route 53 Profiles (April 2024)

  • Bundle DNS configurations into a single, shareable Profile:
    • Resolver rules
    • DNS Firewall rule group associations
    • Private hosted zone associations
  • Apply a Profile to multiple VPCs at once — eliminates per-VPC manual configuration
  • Share Profiles across accounts via AWS RAM

DNS Firewall

  • Launched March 2021 — filter and block outbound DNS queries to malicious domains
  • AWS-managed domain lists (malware, botnet C&C) — no extra cost for the managed lists
  • DNS Firewall Advanced (2023) — detects DNS tunneling and Domain Generation Algorithm (DGA) patterns
EXAM TIP: Route 53 Profiles are the answer for multi-account DNS management at scale. One Profile, many VPCs, shared via RAM.
AWS Docs: Route 53 Developer Guide

6. CloudFront Updates

UPDATED SA Pro Networking

CloudFront Functions (May 2021)

  • Lightweight JavaScript-only functions running at 400+ edge locations
  • Sub-millisecond execution time
  • Viewer request/response only (cannot trigger on origin events)
  • Use cases: URL rewrites, header manipulation, redirects, simple A/B testing
  • Approximately 1/6 the price of Lambda@Edge

CloudFront Functions vs Lambda@Edge

Feature CloudFront Functions Lambda@Edge
Runtime JavaScript only Node.js, Python
Location Edge locations (400+) Regional edge caches (13)
Triggers Viewer request/response only All 4 events (viewer + origin)
Max execution time <1 ms 5s (viewer) / 30s (origin)
Memory 2 MB 128–10,240 MB
Network access No Yes
Cost ~1/6 of Lambda@Edge Higher

Origin Access Control (OAC) — August 2022

  • Replaces Origin Access Identity (OAI) — OAI still works but is considered legacy
  • Supports SSE-KMS encrypted objects (OAI does not)
  • Works in all Regions, including opt-in Regions
  • Supports PUT/DELETE requests to S3 (not just GET)
  • Uses SigV4 for authentication

CloudFront KeyValueStore (November 2023)

  • Global, low-latency key-value data store accessible from CloudFront Functions
  • Use cases: URL redirect maps, feature flags, A/B test config, geo-targeting rules
  • Eliminates hardcoding lookup data into function code

Continuous Deployment (November 2022)

  • Create a staging distribution linked to your production distribution
  • Percentage-based traffic split or header-based routing to the staging config
  • Validate changes on real traffic before promoting to production
EXAM TIP: Know the CloudFront Functions vs Lambda@Edge decision matrix (see table above). OAC is replacing OAI — if a question mentions SSE-KMS + CloudFront, the answer is OAC. KeyValueStore enables dynamic edge configuration without redeploying functions.
AWS Docs: CloudFront Developer Guide

7. Global Accelerator

UPDATED Networking

AWS Global Accelerator uses the AWS global backbone to route traffic to optimal endpoints, improving availability and performance.

Current State

  • 130+ Points of Presence in 95 cities across 53 countries
  • Static anycast IPs — two static IPs per accelerator from independent network zones for fault tolerance. Anycast-routed to nearest PoP.
  • BYOIP support — bring up to two /24 IP ranges for use with Global Accelerator
  • TCP termination at edge — TCP connections terminate at the nearest edge location and are proxied over the AWS backbone, reducing latency and improving throughput

Key Features

  • Standard Accelerators — route traffic to optimal healthy endpoints based on health checks, client location, and configured routing policies (weighted, endpoint groups)
  • Custom Routing Accelerators — deterministic routing to specific EC2 instances rather than health-check-based selection. Designed for use cases where each user must connect to a specific instance:
    • Multiplayer gaming (connect player to specific game server)
    • VoIP / WebRTC (session affinity to media server)
    • Real-time collaboration (route to specific compute instance)
  • Traffic dial — percentage-based controls for gradual regional traffic shifting between endpoint groups. Use for blue/green deployments, failover testing, or regional migration.
  • Endpoint weights — control traffic distribution among endpoints within an endpoint group
  • Cross-region failover — automatic failover to healthy endpoints in other Regions based on health checks
Global Accelerator vs CloudFront: CloudFront is a CDN for HTTP/HTTPS content delivery with caching. Global Accelerator is a network-layer (TCP/UDP) accelerator with no caching — use it for non-HTTP protocols, gaming, IoT, VoIP, or when you need static anycast IP addresses.
AWS Docs: Global Accelerator Developer Guide

8. Transit Gateway Updates

UPDATED SA Pro Networking

Major New Features

  • Security Group Referencing (Sep 2024) — reference security groups across VPCs attached to the same TGW. Massive simplification vs. maintaining CIDR-based rules. Update SGs, not route tables.
  • Flow Logs (Jul 2022) — capture and publish TGW traffic metadata to CloudWatch Logs, S3, or Kinesis Data Firehose
  • Connect Attachments (2021) — enable SD-WAN and third-party network appliance integration:
    • Use GRE tunnels with BGP for dynamic routing between appliances and TGW
    • Up to 5 Gbps per Connect attachment, and 20 Gbps with ECMP (Equal-Cost Multi-Path) — significantly higher throughput than IPsec VPN (1.25 Gbps per tunnel)
    • Use case: connecting SD-WAN appliances (Cisco, VMware SD-WAN, Aruba) to Transit Gateway at higher throughput than Site-to-Site VPN
    • Explicitly listed in the ANS-C01 exam guide under Task Statement 1.5 (hybrid routing solutions)

Recent Enhancements

  • Path MTU Discovery (Nov 2024) — improved handling of large packets across TGW
  • Per-AZ CloudWatch metrics (Nov 2024) — monitor traffic per Availability Zone for granular troubleshooting
  • Multicast support — matured since 2019, supports IGMP and static multicast groups
EXAM TIP: Security Group Referencing is a huge simplification for multi-VPC architectures — know this well. Connect Attachments are the answer for SD-WAN integration. Understand when to use TGW (single-Region hub-spoke) vs Cloud WAN (multi-Region) vs VPC Lattice (service-to-service Layer 7).
AWS Docs: Transit Gateway Guide

9. Network Firewall

UPDATED SA Pro Networking Security

Managed, stateful network firewall and IDS/IPS service using a Suricata-compatible rule engine.

Key Capabilities

  • AWS-managed threat signature rule groups — curated by AWS Threat Intelligence, no extra cost
  • TLS inspection — decrypt, inspect, and re-encrypt traffic in both directions (inbound and outbound)
  • Geographic IP filtering — block or allow traffic by country
  • Forward proxy for outbound traffic — domain-based filtering with SNI inspection
  • Transit Gateway integration for East-West traffic inspection between VPCs
  • Firewall Manager for centralized, cross-account firewall policy management
When to use what:
  • Security Groups — instance-level stateful rules (Layer 4)
  • NACLs — subnet-level stateless rules (Layer 4)
  • AWS WAF — Layer 7 (HTTP/HTTPS) protection at CloudFront, ALB, API Gateway
  • Network Firewall — VPC-level IDS/IPS, deep packet inspection, TLS inspection, domain filtering
EXAM TIP: Know when to use Network Firewall vs WAF vs Security Groups vs NACLs. Network Firewall is for deep packet inspection, IDS/IPS, and TLS decryption. WAF is for Layer 7 web application protection.
AWS Docs: Network Firewall Developer Guide

10. PrivateLink Updates

UPDATED SA Pro Networking

AWS PrivateLink provides private connectivity between VPCs, supported AWS services, and on-premises networks without exposing traffic to the public internet.

Endpoint Types

  • Interface VPC Endpoints — ENI-based endpoints to connect to AWS services (e.g., S3, DynamoDB, STS) and partner/custom services via NLB
  • Gateway Load Balancer Endpoints — route traffic through third-party security appliances (firewalls, IDS/IPS) transparently
  • Resource VPC Endpoints (NEW) — access specific VPC resources (e.g., RDS databases) in other accounts without requiring NLBs. Simplifies cross-account database connectivity.
  • Service Network VPC Endpoints (NEW) — connect a VPC to a VPC Lattice service network via PrivateLink, bridging the two networking models
Resource VPC Endpoints are a significant addition — previously, exposing RDS or other resources via PrivateLink required an NLB in front. Resource endpoints remove that requirement.
AWS Docs: PrivateLink Guide

11. Elastic Load Balancing

UPDATED SA Pro Networking

Application Load Balancer (ALB)

  • mTLS support — mutual TLS authentication at the load balancer
  • Weighted target groups — percentage-based traffic distribution across target groups for canary/blue-green
  • gRPC support — native HTTP/2 and gRPC routing
  • AWS Load Balancer Controller — Kubernetes-native controller for provisioning ALBs and NLBs from Ingress and Service resources

Network Load Balancer (NLB)

  • Security Group support (NEW — 2023) — NLB now supports security groups. Previously, NLBs did not support SGs, requiring CIDR-based rules on targets. Major simplification.
  • TLS/SNI improvements for multi-domain termination

Gateway Load Balancer (GWLB)

  • GENEVE encapsulation — preserves original packet metadata through security appliances
  • Deploy third-party firewalls, IDS/IPS, and security appliances transparently
  • Integrates with PrivateLink for cross-account deployment
EXAM TIP: NLB security group support is new and important — questions about simplifying NLB security rules should point to SG support. GWLB + GENEVE is the pattern for inline security appliance inspection.
AWS Docs: Elastic Load Balancing Guide

12. Amazon API Gateway Updates

Updated SA Pro Networking

Recent Updates

  • REST API Response Streaming (Nov 2025): streaming responses for LLM/GenAI use cases
  • Private Integration with ALB (Nov 2025): direct ALB integration eliminates NLB intermediary requirement for private APIs
  • TLS 1.3 support

API Types Comparison

Type Key Features Pricing
HTTP APIs Simpler, JWT authorizers, OIDC. No usage plans or API keys. $1.00/million requests
REST APIs Full-featured: usage plans, API keys, request/response transformation, caching, WAF integration $3.50/million requests
WebSocket APIs Real-time bidirectional communication $1.00/million messages
EXAM TIP: HTTP APIs vs REST APIs: HTTP = simpler + cheaper (use when possible). REST = full features (use when you need caching, WAF, transformations, usage plans). Response streaming is key for GenAI architectures. The ALB private integration removes a common NLB workaround.

13. AWS Cloud Map

Networking SA Pro

Cloud resource discovery service that lets applications discover services and their locations via DNS or API.

Discovery Modes

  • DNS-based discovery — registers services in Route 53 private hosted zones. Clients use standard DNS resolution.
  • API-based discovery — clients call the Cloud Map API or SDK to discover instances. Supports richer attributes beyond host/port.

Integrations

  • ECS Service Connect — uses Cloud Map under the hood for service-to-service communication within ECS
  • EKS — Kubernetes service discovery can integrate with Cloud Map for cross-platform lookups
Use Case: Dynamic service discovery in containerized architectures where IP addresses and ports change frequently due to scaling, deployments, or task replacements.
AWS Docs: Cloud Map Developer Guide

14. AWS App Mesh — DEPRECATED

Deprecated SA Pro Networking
Officially deprecated. Discontinuation date September 30, 2026. New customers have been unable to onboard since September 24, 2024.

What It Was

  • Service mesh using Envoy sidecar proxies for service-to-service communication
  • Traffic management, observability, and security across microservices

Replacements

  • VPC Lattice — for EKS and cross-VPC service-to-service communication
  • ECS Service Connect — for ECS-specific service-to-service communication
EXAM TIP: If an exam question describes a service mesh pattern, the modern answer is VPC Lattice (not App Mesh). For ECS-specific service-to-service, ECS Service Connect is preferred. App Mesh may still appear as a distractor.

15. Direct Connect Updates

UPDATED Networking

SiteLink (2021)

  • Create a private network-to-network connection between Direct Connect locations via the AWS backbone
  • Connect global offices directly to each other without routing through a VPC
  • Low-latency, private backbone connectivity between on-premises sites

MACsec (802.1AE)

  • Layer 2 encryption on dedicated Direct Connect connections
  • Supports 10 Gbps, 100 Gbps, and 400 Gbps connections
  • Line-rate encryption — no performance penalty
  • Encrypts frames on the wire between your router and the AWS device

400 Gbps Connections

  • Available at select Direct Connect locations for highest-bandwidth requirements

16. IPv6 Updates

UPDATED SA Pro Networking

IPv6 Capabilities

  • IPv6-only subnets — create subnets with no IPv4 CIDR at all
  • Private IPv6:
    • ULA (Unique Local Addresses) — fd00::/8 for internal-only communication
    • Private GUA (Global Unicast Addresses) — use your own globally routable addresses privately
  • IPAM management for both public and private IPv6 pools
  • Most AWS services now support dual-stack or IPv6-only operation
Public IPv4 Charging (February 2024): AWS now charges $0.005/hour per public IPv4 address (~$3.60/month). This applies to all public IPv4 addresses whether attached or idle. This creates a significant financial incentive to migrate to IPv6.
EXAM TIP: Understand IPv6-only subnets, ULA vs GUA, and the public IPv4 cost implications. Questions about cost optimization may now include IPv6 migration as a valid answer.
AWS Docs: IPv6 in VPC

17. Network Manager

UPDATED Networking

Central management and monitoring console for your global network, including Transit Gateway and Cloud WAN resources.

Key Features

  • Route Analyzer — analyze Transit Gateway route tables and verify that routes exist between source and destination
  • Network topology visualization — graphical view of your global network, VPCs, Transit Gateways, VPNs, and Direct Connect

18. AWS Local Zones & Wavelength

Updated SA Pro Networking

Edge Compute Options

  • Local Zones: 33 locations. AWS infrastructure in metro areas for single-digit ms latency. Supports EC2, EBS, ECS, EKS, VPC.
  • Wavelength: 30 zones in 31 cities. AWS compute inside telecom 5G networks for ultra-low latency mobile/edge applications.
  • Outposts: YOUR data center. For data residency, local processing.
EXAM TIP: Decision matrix: CloudFront = CDN caching (static/dynamic content). Local Zones = full AWS services at metro-level latency. Wavelength = 5G edge (mobile apps). Outposts = on-premises (data residency/sovereignty). Global Accelerator = network-layer acceleration without caching.
