4. S3 Default Changes

Silent but Critical S3 Defaults

Behavioral Changes
These defaults changed without fanfare but fundamentally affect how S3 behaves. Exam questions now assume these are in effect. If you learned S3 before 2023, your mental model may be wrong.
When Change Impact
Jan 2023 All new objects auto-encrypted with SSE-S3 No action needed for encryption at rest — it is now always on. SSE-KMS is only needed for CMK control, key rotation policies, or cross-account access patterns.
Apr 2023 Block Public Access enabled by default on new buckets New buckets cannot be made public unless you explicitly disable this. Intentionally public buckets (static sites) require manual override.
Apr 2023 ACLs disabled by default (bucket owner enforced) Object ownership is always the bucket owner. Cross-account uploads no longer need bucket-owner-full-control ACL — the bucket owner automatically gets ownership.
2023 Bucket Keys enabled by default for SSE-KMS Reduces KMS API call costs by up to 99%. Each bucket gets a short-lived key instead of calling KMS per object.
Exam tip: If a question mentions creating a "new S3 bucket" without specifying configuration, assume all four of these defaults are active. Questions about making a bucket public will require explicitly disabling Block Public Access.

5. Pricing Changes to Know

Key Pricing Changes (2021–2026)

Cost Impact
Public IPv4 now costs money. As of February 2024, every public IPv4 address incurs a charge of $0.005/hour ($3.60/month each). This applies to EC2, RDS, ELB, NAT Gateway, and any other service with a public IPv4. Use IPv6, Elastic IPs you actually need, or NAT Gateways strategically.
Change Detail Impact
Public IPv4 Charging $0.005/hour per public IPv4 (Feb 2024) $3.60/month per address. Audit unused EIPs, consider IPv6, use VPC endpoints for private communication.
ElastiCache Valkey 20% cheaper than Redis, 33% cheaper Serverless Valkey (open-source Redis fork) is the cost-optimized choice for new cache deployments.
EBS gp3 Baseline 3,000 IOPS + 125 MB/s included (no extra charge vs gp2) gp3 is almost always cheaper than gp2 for the same performance. Migrate existing gp2 volumes.
Aurora I/O-Optimized Eliminates per-I/O charges Up to 40% savings for I/O-heavy workloads. Predictable pricing without I/O surprises.
Terraform/HCP Pricing Legacy free tier ended Mar 2026; new Free plan covers 500 resources Small teams may need to evaluate HCP costs vs. self-hosted Terraform or OpenTofu.
Kinesis On-Demand Advantage Mode 60%+ cost discount vs standard on-demand (2025) Significant savings for Kinesis Data Streams workloads. Evaluate switching existing streams.
Redshift Serverless Pay only for compute used (RPU-hours), no idle costs Eliminates over-provisioning costs. Ideal for intermittent or unpredictable analytics workloads.
Exam tip: Cost optimization questions now frequently involve IPv4 charges and choosing the right storage tier (gp3 vs gp2, Aurora I/O-Optimized vs Standard). Know when each pricing model is advantageous.
