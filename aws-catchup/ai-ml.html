1. Amazon Bedrock

Amazon Bedrock

NEW - GA Sep 2023 SA Pro ML AI Practitioner

Fully managed generative AI platform — access foundation models (FMs) from multiple providers via a single API. No infrastructure management required. Build, customize, and deploy GenAI applications at scale.

Core Concept: Bedrock is the serverless, API-based gateway to foundation models. You do not manage any underlying infrastructure. Think of it as the "Lambda of GenAI" — you call it, you pay per token, AWS handles the rest.

Foundation Models Available

  • Anthropic Claude — Claude 3.5 Sonnet, Claude 3 Opus, Claude 3 Haiku, Claude 4 family
  • Meta Llama — Llama 3.1, Llama 3.2, Llama 4 family
  • Amazon Nova — Micro, Lite, Pro, Premier, Canvas, Reel (AWS first-party models)
  • Stability AI — Stable Diffusion for image generation
  • Cohere — Command, Embed models for text and embeddings
  • AI21 Labs — Jamba models
  • Mistral AI — Mistral Large, Mixtral models
  • OpenAI GPT — Added 2025/2026, GPT-4o and related models
Docs: Supported foundation models in Bedrock

Bedrock — Agents & AgentCore

Build autonomous AI agents that can break down complex tasks, call APIs, and take multi-step actions on behalf of users.

  • Multi-step task automation — agent decomposes user requests into a sequence of actions
  • API invocation — agents call external APIs via action groups (backed by Lambda or OpenAPI schemas)
  • Memory — persistent session context across conversations for continuity
  • Code interpretation — agent can write and execute code to solve data analysis tasks
  • Multi-agent collaboration — orchestrate multiple specialized agents working together
  • AgentCore — managed infrastructure layer for deploying agents at scale with built-in observability
Docs: Agents for Amazon Bedrock

Bedrock — Knowledge Bases (RAG)

Managed Retrieval-Augmented Generation (RAG) service. Connect your data sources, automatically chunk/embed/index content into a vector store, and retrieve relevant context at query time to ground FM responses in your data.

  • Data source connectors: Amazon S3, SharePoint, Confluence, Google Drive, Web Crawler
  • Vector stores: Managed (Amazon OpenSearch Serverless) or custom (Pinecone, Redis, Aurora PostgreSQL with pgvector, MongoDB Atlas)
  • Smart parsing: Automatic chunking strategies (fixed-size, semantic, hierarchical) with metadata extraction
  • Agentic Retrieval: Knowledge base can be used as a tool by Bedrock Agents for dynamic, context-aware retrieval
  • Pricing: Flat per-query pricing — predictable costs regardless of document volume
RAG Architecture: Data sources → Chunking & Embedding → Vector Store → Query: user prompt + retrieved context → FM generates grounded response. This is the key pattern for reducing hallucination and incorporating private data.
Docs: Knowledge bases for Amazon Bedrock

Bedrock — Guardrails

SA Pro AI Practitioner

Configurable safeguards for responsible AI. Apply content filtering, topic restrictions, and safety policies to any FM — including third-party and self-hosted models. Blocks 88% of harmful content.

Six Safeguard Policies

  1. Content Filters — filter harmful content across categories (hate, insults, sexual, violence, misconduct) with configurable strength levels
  2. Prompt Attack Detection — detect and block prompt injection and jailbreak attempts
  3. Denied Topics — define topics the model must refuse to engage with (e.g., competitor advice, investment recommendations)
  4. PII Redaction — automatically detect and redact/mask personally identifiable information in inputs and outputs
  5. Contextual Grounding — detect hallucinations by checking FM responses against source context and relevance thresholds
  6. Automated Reasoning — uses formal mathematical logic to validate FM outputs against policy rules. 99% accuracy (deterministic, not probabilistic). This is unique to AWS.
Key Feature: Guardrails work with ANY foundation model — not just Bedrock-hosted models. You can apply them to third-party FM APIs and self-hosted models. Also supports cross-account deployment via AWS Organizations.
EXAM TIP: Guardrails is the primary answer for any question about responsible AI at runtime, content moderation for GenAI, or preventing hallucination. Automated Reasoning is a key differentiator — it uses formal logic (deterministic), not AI-based detection (probabilistic).
Docs: Guardrails for Amazon Bedrock

Bedrock — Model Evaluation

Compare foundation models to select the best one for your use case using automatic and human evaluation methods.

  • Automatic evaluation metrics: BERT-Score (semantic similarity), ROUGE (summarization quality), F1 Score (classification accuracy), Exact Match
  • Human evaluation: Set up human review workflows with custom criteria and rating scales
  • Side-by-side comparison: Evaluate multiple models on the same dataset simultaneously
  • Custom metrics: Define domain-specific evaluation criteria
Docs: Model evaluation in Amazon Bedrock

Bedrock — Fine-Tuning & Custom Models

Customize foundation models with your own data for domain-specific performance improvements.

  • Continued Pre-training: Train a model on large volumes of unlabeled domain data to improve domain knowledge
  • Fine-tuning: Train on labeled prompt-response pairs to improve performance on specific tasks
  • Model Import: Bring your own custom-trained models (e.g., from SageMaker or Hugging Face) into Bedrock for serverless deployment
Docs: Custom models in Amazon Bedrock

Bedrock — Model Distillation

NEW

Teacher-student model optimization. Use a large, capable model (teacher) to train a smaller, faster model (student) that mimics the teacher's performance for your specific use case.

  • Up to 500% faster inference compared to the teacher model
  • Up to 75% cheaper per request
  • Less than 2% accuracy loss vs. the teacher model on target tasks
  • Automated process — provide sample prompts, Bedrock handles the rest
EXAM TIP: Distillation is a cost optimization strategy. When the exam asks about reducing GenAI costs while maintaining quality, distillation is a key answer alongside prompt routing and caching.
Docs: Model distillation in Amazon Bedrock

Bedrock — Intelligent Prompt Routing

NEW

Automatically routes each prompt to the optimal model within a model family based on complexity. Simple prompts go to smaller/cheaper models; complex prompts go to larger/more capable models.

  • Up to 30% cost reduction with no quality degradation
  • Works within a model family (e.g., Claude Haiku vs. Claude Sonnet)
  • No code changes required — configure the routing profile and call the endpoint
Docs: Intelligent prompt routing

Bedrock — Prompt Caching

NEW

Cache frequently used prompt prefixes (system prompts, few-shot examples, long context documents) to avoid reprocessing them on every request.

  • Up to 90% cost reduction on cached tokens
  • Up to 85% latency reduction on cache hits
  • Ideal for applications with shared system prompts or repeated context across requests
  • Configurable cache TTL
Docs: Prompt caching in Amazon Bedrock

Bedrock — Pricing Model

Pricing Tier How It Works Best For
On-Demand Pay per input/output token, no commitment Variable workloads, experimentation
Batch Submit batch jobs, 50% lower cost than on-demand Non-real-time processing, bulk analysis
Provisioned Throughput Reserved model units for guaranteed capacity Predictable, high-volume production workloads

Service Tiers (for on-demand)

  • Standard — default tier, best-effort throughput
  • Flex — lower cost, variable latency (queued processing)
  • Priority — higher throughput guarantees during peak demand
  • Reserved — committed throughput with term discounts
EXAM TIP — Cost Optimization Trio: Know the three Bedrock cost optimization features: Model Distillation (smaller model, same quality), Intelligent Prompt Routing (right-size per request), and Prompt Caching (avoid reprocessing). Exam loves asking about cost-optimized GenAI architectures.

2. Amazon Q

Amazon Q

NEW - GA April 2024 SA Pro DevOps

AWS's AI-powered assistant family. Amazon Q comes in four variants, each purpose-built for a different audience. This is a critical service to understand — the exam expects you to know which variant to recommend for each scenario.

Q Developer

  • AI-powered code generation, completion, and debugging in IDEs (VS Code, JetBrains, CLI)
  • Security scanning — identifies vulnerabilities in generated and existing code
  • .NET and Java modernization — automated code transformation to modern frameworks (e.g., .NET Framework to .NET Core, Java 8 to Java 17)
  • AWS console expert — ask questions about AWS services and get contextual answers
  • Replaced Amazon CodeWhisperer (CodeWhisperer was rebranded to Q Developer)
Docs: Amazon Q Developer

Q Business

  • Enterprise knowledge assistant — connects to your organization's data and provides answers grounded in internal content
  • 50+ built-in actions — integrates with Jira, Salesforce, ServiceNow, Slack, Microsoft Teams, and more
  • Q Apps — no-code app builder powered by natural language; create lightweight internal tools from conversation
  • Accessible from browser, Slack, and Microsoft Teams
  • IAM Identity Center integration for user management and access control
Docs: Amazon Q Business

Q in QuickSight

  • Natural language to dashboards and visualizations
  • Ask questions in plain English, get charts, graphs, and insights
  • Integrated directly into Amazon QuickSight BI platform
Docs: Q in QuickSight

Q in Connect

  • Contact center agent assist for Amazon Connect
  • Real-time recommendations and answers during customer calls
  • Surfaces knowledge base articles, suggested responses, and next-best-action guidance
Docs: Q in Connect
EXAM TIP: Know all 4 variants. Q Developer = code + modernization scenarios (.NET/Java). Q Business = enterprise search / internal knowledge assistant. Q in QuickSight = BI/analytics. Q in Connect = contact center. When the exam says "AI-powered code transformation," the answer is Q Developer, not CodeWhisperer.

3. Amazon SageMaker

Amazon SageMaker

MAJOR UPDATE ML SA Pro

Full lifecycle ML platform that has undergone massive restructuring since 2021. SageMaker is now an umbrella covering everything from no-code ML to distributed training at scale.

SageMaker Unified Studio

NEW - re:Invent 2024

A single interface that combines ML, data engineering, and analytics workflows. Previously these were separate tools (SageMaker Studio, DataZone, EMR Studio, Athena, etc.).

  • Built on Amazon DataZone for data governance and cataloging
  • Integrated IDEs: JupyterLab, Code Editor (VS Code-based), RStudio
  • SageMaker Catalog — unified governance layer for discovering and managing ML models, datasets, and features across teams
  • Combines previously siloed SageMaker Studio, SageMaker Domain, and DataZone into one experience
Docs: SageMaker Unified Studio

SageMaker JumpStart

UPDATED

ML hub with foundation models, pre-trained models, and solution templates from multiple providers. More than just GenAI — includes traditional ML models, computer vision, NLP, and tabular models.

  • Access FMs from Hugging Face, Meta, Stability AI, and others — deploy to SageMaker endpoints
  • Fine-tune models within SageMaker with full control over training infrastructure
  • Pre-built solution templates for common ML use cases
  • Full infrastructure control — choose instance types, configure autoscaling, set up A/B testing
JumpStart vs. Bedrock — Key Distinction:
SageMaker JumpStart = broader ML hub with full infrastructure control. You manage endpoints, scaling, and deployment. Supports both GenAI and traditional ML.
Amazon Bedrock = serverless, API-based FM access. No infrastructure to manage. GenAI only.
When the exam asks about "more control over model deployment" or "custom inference endpoints," the answer is JumpStart. When it asks about "simplest way to access FMs" or "fully managed GenAI," the answer is Bedrock.

SageMaker Canvas

UPDATED

No-code ML for business analysts. Build, train, and deploy ML models using a visual, point-and-click interface — no coding or ML expertise required.

  • AutoML — automatically selects algorithms, tunes hyperparameters, and trains models
  • Connects to 50+ data sources including S3, Redshift, Snowflake, SaaS applications
  • Supports 300+ built-in analyses for classification, regression, forecasting, NLP, and computer vision
  • Replaces Amazon Forecast for time-series forecasting use cases (Forecast is deprecated)
  • Chat-based model building with natural language
Amazon Forecast is DEPRECATED. It is no longer available to new customers. SageMaker Canvas is the recommended replacement for all time-series forecasting use cases. Existing Forecast users should migrate.

SageMaker HyperPod

NEW ML

Managed distributed training infrastructure purpose-built for training large foundation models at scale. Handles the hard parts of distributed training automatically.

  • Checkpointless training — innovative approach that reduces checkpoint overhead for long-running training jobs
  • Automatic fault detection and repair — detects hardware failures and automatically replaces faulty nodes without restarting the entire training job
  • Up to 40% reduction in training cost and time compared to self-managed distributed training
  • Supports Slurm and Amazon EKS orchestrators
  • Integrates with AWS Trainium and NVIDIA GPU instances
Docs: SageMaker HyperPod

SageMaker Clarify

UPDATED ML AI Practitioner

Bias detection and model explainability for both traditional ML models and foundation models.

  • Pre-training bias detection — analyze training data for imbalances before training
  • Post-training bias detection — evaluate model predictions for disparate impact across groups
  • FM stereotype evaluation — test foundation models for stereotypical associations and biases
  • Explainability — SHAP-based feature attribution for model predictions

4. AWS Trainium & Inferentia

AWS Trainium & Inferentia

NEW/UPDATED ML

AWS-designed custom silicon for ML workloads. Cost-optimized alternatives to NVIDIA GPUs for training and inference.

Trainium (Training)

Purpose-built chips for training ML models at scale.

Generation Instance Key Specs
Trainium (Gen1) Trn1 First generation, up to 16 chips per instance
Trainium2 (Gen2) Trn2 4x training performance vs. Trn1, improved networking
Trainium3 (Latest) TBD 8 NeuronCore engines, 144GB HBM3e memory per chip

UltraServers: Aggregated configurations with up to 144 Trainium chips for extremely large training jobs.

Inferentia (Inference)

Purpose-built chips for running ML inference at low cost and high throughput.

Generation Instance Key Specs
Inferentia (Gen1) Inf1 4 NeuronCores per chip, cost-effective baseline inference
Inferentia2 (Gen2) Inf2 32GB HBM per chip, 4x throughput, 10x lower latency vs. Inf1

AWS Neuron SDK

Unified SDK for both Trainium and Inferentia. Provides compiler, runtime, profiling, and framework integration (PyTorch, TensorFlow, JAX).

Key Concept: Trainium = training, Inferentia = inference. Both are cost-optimization strategies vs. GPU instances (p4d, p5, g5, etc.). The exam tests whether you know which chip is for which workload, and when to recommend custom silicon over GPUs.

Amazon Lex V2

MAJOR UPDATE SA Pro

Conversational AI service for building chatbots and voice bots. V2 is a complete redesign.

  • Redesigned API — not backward compatible with V1. Separate console, SDK calls, and resource model
  • Assisted NLU using LLMs — leverage large language models to improve intent recognition and slot filling without extensive training data
  • Conditional branching — build complex conversation flows directly in Lex without needing Lambda functions for routing logic

  • Visual conversation builder — drag-and-drop UI for designing conversation flows
  • GenAI-powered bot generation — describe your bot in natural language, Lex generates the intents and slots
Lex V1 is deprecated. All new development should use Lex V2. The APIs are not interchangeable. Exam questions about Lex will be V2-focused.

7. Responsible AI

Responsible AI

SA Pro ML AI Practitioner

AWS Responsible AI Framework — a structured approach to building trustworthy AI systems. This is a major exam topic, especially for the AI Practitioner certification.

Eight Dimensions of Responsible AI

Dimension Description Key AWS Tool
1. Fairness Ensure AI does not discriminate against groups; equitable outcomes SageMaker Clarify
2. Explainability Understand why a model made a particular decision SageMaker Clarify (SHAP)
3. Privacy & Security Protect data and model access; comply with regulations Guardrails (PII), IAM, KMS
4. Safety Prevent harmful outputs and unintended consequences Bedrock Guardrails
5. Controllability Human oversight and ability to intervene in AI decisions Guardrails, Ground Truth
6. Veracity & Robustness Accurate, reliable outputs; resilience to adversarial inputs Guardrails (Automated Reasoning, Contextual Grounding)
7. Governance Organizational policies, processes, and accountability for AI SageMaker Model Cards, SageMaker Model Registry
8. Transparency Clear communication about AI capabilities and limitations to users SageMaker Model Cards, documentation

Key AWS Tools for Responsible AI

  • Bedrock Guardrails — runtime safety: content filtering, PII redaction, prompt attack detection, hallucination detection, Automated Reasoning
  • SageMaker Clarify — pre/post-training bias detection, SHAP-based explainability, FM stereotype evaluation
  • SageMaker Model Cards — structured documentation of model purpose, training data, performance, limitations, and intended use
  • SageMaker Model Monitor — production monitoring for data drift, model quality degradation, bias drift, and feature attribution drift
  • SageMaker Ground Truth — human-in-the-loop labeling and review; enables human feedback loops for model improvement
EXAM TIP (AI Practitioner): This topic is VERY HIGH priority for AIF-C01. Know all 8 dimensions and which AWS tool maps to each. Key distinction: Automated Reasoning in Guardrails uses formal mathematical logic (deterministic, provably correct) — it is NOT an AI/ML-based approach. This is different from Contextual Grounding, which uses the FM itself to check factuality. Automated Reasoning achieves 99% accuracy because it is logic-based, not probabilistic.

8. Key Architectural Patterns

Key AI/ML Architectural Patterns

SA Pro ML

Common architecture patterns that appear in exam scenarios. Know the components, data flow, and when to use each pattern.

1. Retrieval-Augmented Generation (RAG)

Pattern: S3 / data sources → Bedrock Knowledge Bases (or Kendra) → chunking & embedding → vector store → user query + retrieved context → FM generates grounded response

When to use: Ground FM responses in private/current data. Reduce hallucination. No model retraining needed.

2. Contact Center AI

Pattern: Amazon Connect → Lex V2 (conversation) → Q in Connect (agent assist) → Transcribe Call Analytics (post-call analysis)

When to use: Intelligent contact center with real-time agent assistance and automated post-call insights.

3. Document Processing Pipeline

Pattern: S3 (document upload) → Textract (text/data extraction) → Comprehend (entity/PII detection) → Lambda (business logic) → DynamoDB (structured storage)

When to use: Automated processing of invoices, forms, medical records, legal documents at scale.

4. Content Moderation Pipeline

Pattern: Content upload → Rekognition (image/video moderation) + Comprehend (text moderation) + Bedrock Guardrails (GenAI content moderation)

When to use: User-generated content platforms, social media, marketplace listings.

5. ML Training at Scale

Pattern: SageMaker HyperPod + Trainium instances (training) → trained model → Inferentia instances (inference deployment)

When to use: Training large custom models or fine-tuning FMs. Cost optimization over GPU instances.

6. No-Code ML

Pattern: Business data → SageMaker Canvas (AutoML, visual interface) → predictions / forecasts

When to use: Business analysts needing ML predictions without data science expertise. Replaces Amazon Forecast for time-series.

7. Cost-Optimized GenAI

Pattern: Amazon Bedrock + Model Distillation (smaller student model) + Intelligent Prompt Routing (right-size per request) + Prompt Caching (avoid reprocessing)

When to use: Production GenAI workloads where cost optimization is critical without sacrificing quality.

8. Enterprise GenAI Assistant

Pattern: Amazon Q Business + Kendra (or Bedrock Knowledge Bases) + data connectors (SharePoint, Confluence, S3) → enterprise knowledge assistant

When to use: Internal enterprise search and Q&A over organizational knowledge bases with access control.

9. Responsible AI Pipeline

Pattern: SageMaker Clarify (pre-training bias check) → model training → Clarify (post-training bias check) → Bedrock Guardrails (runtime safety) → SageMaker Model Monitor (production drift detection)

When to use: Any regulated or high-stakes AI deployment requiring fairness, explainability, and ongoing monitoring.
